OperationDefinition: MergePatient

Official URL: http://hl7.org/fhir/ca-bc/client/OperationDefinition/bc-patient-merge Version: 0.1.0
Draft as of 2022-03-07 Computable Name: MergePatient

​Merge Patient operation needs the Patient.link.other field populated with an Identifier so the non-survivor Patients can be located. There may be more than one set of Identifiers and each will be merged (if they represent more than a single non-surviving Patient). See an example of linking the non-survivors here.

MergePatient

OPERATION: MergePatient

The official URL for this operation definition is: 

http://hl7.org/fhir/ca-bc/client/OperationDefinition/bc-patient-merge

URL: [base]/Patient/$MergePatient

Input parameters Profile: BCPatientBusinessParameters

Parameters

UseNameCardinalityTypeBindingDocumentation
INmessageId1..1string

Unique message ID.

INmessageDate1..1string

Message date and time.

INpatient1..1Patient

The Patient.

INenterer0..1Identifier

UserId of sender.

INsender1..1Identifier

Source of request

Notes:

The response to a MergePatient Operation is a Bundle of type collection. The Bundle includes

  • a Patient resource when the interaction was successful for the surviving Patient
  • a OperationOutcome resource where errors and warnings are returned
  • a Parameters resource (specifically BCMetadataParameters)

If this is an asynchronous merge then the BCMetadataParameters must contain the request MessageID in the requestMessageId field.

See Merge Patient response
